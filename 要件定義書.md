# 要件定義書：三日坊主バスター

## 1. サービス概要

| 項目 | 内容 |
|------|------|
| サービス名 | 三日坊主バスター |
| 種別 | Webアプリ（小規模・1〜3ページ） |
| ターゲット | 個人（一般消費者） |
| コンセプト | 習慣を登録し、毎日チェックイン。達成したら褒めてくれる。未達成が続いたらデポジット（擬似）が寄付される仕組みで三日坊主を防止する |

## 2. 解決する課題

- 習慣を始めても続かない（三日坊主問題）
- モチベーション維持の仕組みがない
- 「やらなかったら損をする」というペナルティ設計で継続を促進

## 3. 機能要件（MVP）

### 3.1 習慣の登録・管理
- 習慣名を入力して登録（例：「走る」「料理を作る」「英会話1時間」）
- 習慣の編集・削除
- チャレンジ期間の設定（7日 / 14日 / 30日 / カスタム）
- デポジット金額の設定（擬似。例：1,000円 / 3,000円 / 5,000円 / カスタム）

### 3.2 毎日のチェックイン
- 今日の習慣一覧を表示
- 各習慣に「達成!」ボタンを配置
- 達成時：褒めメッセージをランダム表示（例：「すごい!」「がんばったね!」「最高!」）
- 1日の終わりに未チェックの習慣は「未達成」扱い

### 3.3 連続達成・ストリーク表示
- 連続達成日数をカウント・表示
- マイルストーン表示（3日・7日・14日・30日など）

### 3.4 デポジット・ペナルティ機能（モック）
- チャレンジ期間終了時に達成率を計算
- 達成率が基準（例：80%）未満の場合「デポジットが寄付されました」と表示
- 達成率が基準以上の場合「デポジットを守りました!」と表示
- ※初期リリースでは実際の決済は行わない（表示のみ）

## 4. 非機能要件

| 項目 | 内容 |
|------|------|
| デザイン | ミニマル・モダン |
| レスポンシブ | スマホ・PC両対応 |
| データ保存 | localStorage（初期）→ 将来的にアカウント機能追加 |
| 認証 | 初期リリースでは不要 |
| ホスティング | GitHub Pages（静的サイト） |

## 5. 技術スタック

| レイヤー | 技術 |
|----------|------|
| 構造 | HTML5 |
| スタイル | CSS3（カスタムプロパティ活用） |
| ロジック | Vanilla JavaScript（ES6+） |
| データ保存 | localStorage |
| ホスティング | GitHub Pages |
| バージョン管理 | Git / GitHub |

※フレームワーク不使用。軽量・シンプル・学習コスト最小を優先。

## 6. 画面構成

### P1: ダッシュボード（メイン画面）
- 今日の日付
- 登録済み習慣の一覧（チェックイン状態付き）
- 各習慣の連続達成日数
- 「+習慣を追加」ボタン

### P2: 習慣登録・編集モーダル
- 習慣名の入力
- チャレンジ期間の選択
- デポジット金額の設定
- 保存 / キャンセルボタン

### P3: チャレンジ結果画面
- 期間終了時の達成率表示
- デポジット結果（守れた / 寄付された）
- 次のチャレンジへの導線

## 7. 将来拡張（初期リリース対象外）

- ユーザー認証（Googleログイン等）
- クラウドデータ同期
- 実決済によるデポジット機能（Stripe連携）
- SNSシェア機能
- バッジ・称号システム
- アニメーション演出（紙吹雪エフェクト等）
- プッシュ通知によるリマインダー
